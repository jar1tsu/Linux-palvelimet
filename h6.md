## h6 Salataampa

#### x) Let`s Encrypt 2024: https://letsencrypt.org/how-it-works/

- ACME‑asiakas todistaa domainin hallinnan (DNS‑tietue tai HTTP‑tiedosto)
- CA (Certificate Authority) tarkistaa haasteen useista verkon suunnista → valtuuttaa asiakkaan hallitsemaan domainin sertifikaatteja
- Asiakas lähettää CSR‑pyynnön, CA tarkistaa allekirjoitukset ja myöntää sertifikaatin (julkaistaan CT‑lokissa)
- Uusiminen = sama prosessi kuin myöntäminen
- Revokointi = valtuutettu asiakas allekirjoittaa perumispyynnön, CA julkaisee sen CRL‑listaan

#### x) The Apache Software Foundation 2025: Apache HTTP Server Version 2.4 [Official] Documentation: SSL/TLS Strong Encryption: How-To: Basic Configuration Example

##### Basic Configuration:

- SSL-moduuli päälle, portti 443 kuunteluun.
- VirtualHost sisältää: SSLEngine päällä sekä polut sertifikaattiin ja avaimiin
- Certbot tekee nämä automaattisesti

##### OCSP Stapling:

- Palvelin liittää selaimelle valmiiksi OCSP‑vastauksen → nopeampi ja yksityisempi
- Perusasetukset: SSLUseStapling päällä, SSLStaplingCache …
- Vaatii toimivan OCSP-osoitteen ja riittävän välimuistin

##### Client Authentication:

- Voidaan vaatia asiakassertifikaatti koko sivustolla tai vain tietyssä polussa
- Tunnistusta voi rajata DN‑arvojen perusteella (esim. organisaatio)
- Mahdollista yhdistää: HTTPS + vahva salaus + salasana tai sertifikaatti

##### Logging

- mod_ssl voi logittaa tarkasti: tasoa voi säätää moduulikohtaisesti
